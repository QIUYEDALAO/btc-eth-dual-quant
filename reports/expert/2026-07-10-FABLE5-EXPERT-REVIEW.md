## 2. Q1:Sharpe 口径审查 —— 不同义,且回撤字段也有同类问题

### 事实(Freqtrade 2026.6 源码)

回测报告的 Sharpe 来自 `calculate_sharpe(trades, min_date, max_date, starting_balance)`:

```
total_profit = trades["profit_abs"] / starting_balance      # 逐笔收益,除以期初资金(非滚动权益)
expected_returns_mean = total_profit.sum() / days_period     # 分子:按日历天摊薄的"日均收益"
up_stdev = np.std(total_profit)                              # 分母:逐笔收益的标准差(ddof=0)
sharpe = mean / std * sqrt(365)
```

分子单位是"每日",分母单位是"每笔",再按日频年化。它不是任何标准意义上的年化 Sharpe:对低频、多数时间空仓的策略,分子被空仓天数摊薄而分母保持逐笔量级,数值被系统性压向 0——M1C 的 0.1146 即由此而来。M1B 报告的 9.40 则来自另一套定义(侧车的"年化收益 ÷ 年化波动")且仅有 2 个 OOS 周期:一个被口径压低、一个被小样本抬高,两个数都不是可用证据。Freqtrade 自身另有 `calculate_sharpe_from_balance`(日频重采样余额的 pct_change,正规日频 Sharpe),但它服务于 live/dry-run 余额快照,**不进入回测汇总输出**。

回撤字段问题:回测的账户回撤来自 `calculate_max_drawdown(trades, value_col="profit_abs", relative=False)`——只在**平仓时点**对已实现利润累计曲线采样,且 `relative=False` 时返回的是"绝对(USDT)最深回撤点上的相对回撤"。两个后果:持仓期间的 MTM 回撤不可见;报告的相对回撤未必是最大相对回撤。你们的 base-full 报告值 15.7711% 即为 2024 年亏损段(绝对额最大)上的相对值;而同一条已实现曲线上,2018 年两笔止损造成的相对回撤是 **19.21%**,已超 15% 门槛,但未被报告字段捕捉。

另外一致性问题:M1B 侧车的 sharpe=7.15 是"年化收益 ÷ 年化波动"的第三种定义。也就是说,"OOS Sharpe ≥ 1.0"这一 Gate 目前在 M1A/M1B/M1C 上消费的是三种互不可比的量。

### 建议的固定标准(预注册,一次定死)

全项目唯一风险口径:**日频 MTM 账户权益曲线**。定义:D1 账户总权益 = 现金 + 持仓清算价值(现货收盘 × 费率因子),UTC 每日收盘采样;D2 算术日收益,空仓日计 0,一年 365 天;D3 `Sharpe_ann = mean(r)/std(r, ddof=1) × √365`,rf=0;D4 MaxDD 与 Sortino/Calmar 全部取自同一条曲线;D5 OOS 指标用 OOS 独立回测(fresh capital)的曲线,维持现口径;D6 Freqtrade 汇总字段仅作交叉参考,不作 Gate 输入。实现方式不改 Freqtrade:由 backtest export 的 trades + 金数据集 K 线独立构建曲线(`m1c_recompute.py` 即模板,全样本版本可直接在你们的数据缓存上运行)。

## 3. Q2:样本量 Gate —— 保留,但补一个与频率无关的统计层

80/20 作为硬门槛,回答的是"可运营性"而非"统计显著性":逐笔统计(胜率、盈亏比、删最佳三笔、成本敏感)在 n<20 时没有意义,自动化后的监控也需要在合理日历时间内积累可检验事件。在 ~9 年历史上,80 笔意味着设计频率 ≥ ~9 笔/年——这实际上是一个频率下限。我的判断:**保留 80/20,并诚实承认其后果**——它按构造排除了持仓期 > 数周的策略;这是组合设计决定,不是需要修补的统计缺陷。M1A/M1B/M1C 三次失败中有两次(M1B 13 周期、M1C 31 笔)在写代码之前就能从"频率 × 历史长度"算出来,这正是 Q4 诊断要拦截的。

但仅有笔数门槛不充分(高频而无边际的策略可以轻松凑笔数),需再加一个**与时间尺度无关、全候选统一**的统计层,建议:OOS 日收益的 PSR(SR*=0) ≥ 0.95(Bailey–López de Prado,按观测日数 n 计算,偏度/峰度自动惩罚肥尾稀疏收益);OOS ≥ 540 个日历日;样本窗口须覆盖至少一个预定义压力段(标的自身回撤 ≥30%)。PSR 以"日"为观测单位,对任何持仓周期一视同仁,满足你们"不能只为 M1C 放宽"的要求——事实上 M1C 修正后 PSR=0.90,在新层下照样失败,所以这不是后门。

多重检验必须计账:同一段历史只有一份,反复提出候选会使"最优者"的表观 Sharpe 虚高。建立试错台账(每个已开封 OOS 或已做先导研究的候选族记一次),随台账规模用 DSR(Deflated Sharpe Ratio)或等效方式上调 SR*。这是"避免在同一历史上反复试错"在数学上唯一诚实的形式——试错无法禁止,只能收费。

功效告诫(写进候选诊断):2.67 年日频数据的 Sharpe 标准误 ≈0.61,95% CI 宽约 ±1.2。即便真 Sharpe=1 的策略,在这个窗口长度下也常无法以 95% 置信确认为正。含义:任何候选的"通过"都主要证明它值得进入下一阶段(更长 OOS、90 天 dry-run 证据),而不是证明边际存在;Gate 应被理解为分层过滤器,不是显著性证书。顺带:Freqtrade 2026.6 已内置 `calculate_p_value`(逐笔 t 检验;M1C OOS 15 笔 t=1.46,不显著),可作为免费的交叉诊断输出。

## 4. Q3:数据权威规则 —— 金数据集 + 冲突隔离,而非二选一

你们审计发现的三类事实(月 ZIP 缺行可由日 ZIP 精确补回;ZIP 与 REST 并存但数值不同;月/日 ZIP 相互一致而与 REST 不同)说明 Binance 的 REST 数据库存在事后修订,而 ZIP 是发布即冻结的归档。审计的目标不应是"让两个官方来源一致"(不可能),而是"固定一个可逐字节复现的数据版本,并证明结论对残余分歧不敏感"。建议规则,全部先于任何未来回测注册:

**权威序(构建 golden dataset v1)**:R1 官方月 ZIP 为基底(不可变、带 CHECKSUM、第三方可复算);R2 官方日 ZIP 仅用于补月 ZIP 缺行(你们已验证 936 行精确补回);R3 REST 仅用于补两级 ZIP 均无的行,且必须一次性快照进 append-only 存储、记录抓取 UTC 与哈希——规则中引用的"REST"永远指该快照,不指活接口;R4 双源并存且不等时,金数据集保留 ZIP 值,同时把冲突写入冲突表(时间戳、字段、双值、|Δ|)。1d 及更粗粒度一律从 golden 1h 单一路径派生,禁止另行下载 1d 归档形成第三套版本。清单(symbol × timeframe × 月 → 来源、行数、SHA256、抓取时刻、补丁列表)+ 冲突表 + 隔离表构成数据版本;任何人重建必须逐哈希一致。

**敏感性与隔离(对称、不可挑数)**:每个候选在 golden(ZIP 优先)与 REST-快照优先两个变体上各跑一遍完整 Gate。任何 Gate 判定翻转 → 涉事月份进入隔离表,从**所有**策略的 IS/OOS 中剔除,直至第三方仲裁;判定全部一致且冲突 bar 的 |Δclose|/close ≤ 预设 ε(建议 10bp)→ 记录后放行。规则双侧对称、先于结果注册,结构上不可能被用于挑选有利数据源。仲裁源预先点名且仅作仲裁:Coinbase/Kraken/Bitstamp(现货)、OKX/Bybit(合约)或商业供应商,以逐 bar 对数收益比对判断哪一侧异常。

**状态语义**:M0 从 `blocked` 改为 `audit_complete_with_quarantine`。"不明确权威版本则不得申请 M2"于是变为可满足的条件:金数据集 v1 已定版 + 该策略敏感性双跑一致 + 不依赖隔离窗口。就现状而言:已知 spot 冲突(2020-12-21、2021-04-23)是 1h 级、非周日、不落在 M1C 决策截面上,预计敏感性双跑一致——但应按上述规则正式执行一次,作为流程的首个范例。2026-06 的 REST-only 时间戳(日 ZIP 尚未发布,404)属发布时滞,不是修订,建议在审计中单列为 `pending_archive` 而非 blocker。

## 5. Q4:候选可行性诊断 —— 同意,给出模板

同意"先诊断、再预注册、后写代码"。诊断以一页纸回答八项,任一不达即不进入实现;诊断本身计入试错台账:

| 字段 | 要求 |
|---|---|
| 经济假设 | 一句话:谁在付钱、为什么会持续(行为偏差/结构性流动/风险溢价/约束套利) |
| 样本预算 | 设计频率 × 可用历史 ≥ 1.5×Gate(全样本 ≥120 笔、OOS ≥30 笔的设计冗余) |
| 成本可行性 | 预期单笔毛边际 ≥ 3× 双边成本(x2 档 0.6%,事件策略再加压力滑点档) |
| 执行现实 | 信号时标 vs 日线粒度;事件期成交假设;仅现货多头可表达 |
| 独立性 | 与已冻结策略的信号相关性、与其收益流的日收益相关性目标 <0.3 |
| 数据契约 | golden dataset 覆盖;不依赖隔离窗口 |
| 预注册 | 全套 Gate(含 D1–D6 口径、PSR 层)、kill 条件、OOS 封存边界,一次开封 |
| 台账 | 登记本候选与全部先导研究;DSR 参数随台账更新 |

允许一个"先导功效测试":仅 IS、向量化、理想成交,只回答频率与毛利量级是否达标,不产出可比业绩、不调参,并记入台账。M1B(13 周期)与 M1C(31 笔)都会在"样本预算"一行被纸面否决——两轮完整工程的成本本可省下,这是该模板的直接证据。

## 6. Q5:下一候选的经济逻辑

首选:**恐慌回撤后的流动性提供(短周期现货均值回归)**。经济来源:强平级联与止损潮制造强制卖出,价格短时超调,极端日的买盘赚取回弹溢价——收益来自向受迫流动性提供服务,与 M1A(趋势延续)、M1B(carry)、M1C(相对强弱轮动)在收益状态上近似正交:它在震荡与恐慌后期盈利、在单边下行初段亏损。可预注册的骨架(参数留待设计冻结,不在此定):触发 = 1–3 日收益 z 分数低于阈值,或距 N 日高点回撤超过 k× 已实现波动;入场 = 下一日开盘(与既有时间语义一致);退出 = 固定时间退出(3–10 日)或回归目标,**不用价格止损**(对均值回归,价格止损系统性截断回归本身;风险以仓位上限 + 时间退出承担);仓位 ≤50% 权益,BTC/ETH 事件独立计数。样本预算:中等阈值下 BTC+ETH 合并历史 ~10–20 事件/年 → 9 年 90–180 笔、OOS 25–50 笔,**操作层门槛按设计可达**——这正是前三条路线全部缺失的性质。成本预算:目标单笔毛边际 2–6% vs 0.3–0.6% 双边成本,达标。特别要求:事件日执行恶化是该假设的固有风险(边际恰好出现在流动性最差时),预注册第三档成本压力(事件日每边 +0.10–0.25%);左尾场景(2020-03、2022 连续下跌型)必须出现在 IS 压力段清单中。

次选:**资金费/基差极端值作为现货反向情绪信号**。与 M1B 的本质区别:不收租、不建两腿,资金费仅作条件信息,交易纯现货单腿,Freqtrade 原生可表达;经济来源是持仓拥挤度(极端正资金费 = 多头拥挤 → 规避,极端负 = 投降 → 做多现货)。数据已在 M0 审计范围内,复用即可。频率 ~20–60 事件/9 年,处于样本预算边缘,诊断时要诚实计算;若首选先行,此候选须通过与其的独立性检查(投降事件两者重叠)。

明确不建议:时段/星期季节性(单次边际 5–20bp,被 30bp/边成本直接淹没,且属全社区过度挖掘区);波动率目标/波动压缩突破(在单资产 long-only 下退化为另一种 regime 轮动,频率同样不足,近似 M1C 换皮);ETH/BTC 比值均值回归(比值长期非平稳、无空头腿无法对冲漂移)。

范围旁注,供治理层记录:两个高度相关资产、只做多、9 年历史,是统计上非常小的假设宇宙,候选的先验成功率本来就低。若上述两条经诊断或验证亦失败,理性的下一步不是第六个假设,而是经新 ADR 扩大现货宇宙(如 Binance 流动性前 N 现货对)做横截面策略——独立观测数量级增长,样本 Gate 自然可达。此为范围决策,不影响当前候选。

## 7. Q6:M1B 定位
